阿里的一些Java编程规范解读

Posted on 2020-08-11 Edited on 2020-08-20 In Technical , Java , 基础

参考文章：

  1. 阿里 《Java开发手册解读》

tips:

  1. 禁止使用Apache BeanUtils 进行属性copy。 => 性能问题；
  2. 日期格式化年份使用y, 不能使用Y。 => 国际标准问题，边界情况Y会导致年份错误；
  3. 三目运算符注意自动拆箱导致的空指针问题。 => 基本类型的引用类型自动拆箱的问题，会导致抛出NPE；
  4. 建议设置初始化HashMap的大小，值为 期望大小/扩展因子(默认0.75) + 1。 => HashMap重建hash树性能损耗；
  5. 禁止使用Executors创建线程池。 => 默认的都可能会导致OOM问题；
  6. 谨慎使用ArrayList 中的 sublist方法 => 返回结果是ArrayList 类型的内部类，是它的一个视图，不是一个新的List，因此使用不方便容易出错；
  7. 不建议在for 循环中使用 “+” 进行字符串拼接 => “+” 反编译会变成”StringBuilder”，线程不安全；
  8. 禁止在forEach循环中使用add/remove操作 => 可能会报错。forEach语法糖反编译，其实使用了iterator; add/remove是list的方法, list会报并发修改错误；
  9. 禁止工程师直接使用日志系统(Log4j、Log back) 中的API => 外观设计模式，使用日志门面，主要一套通用的API，用来屏蔽各个日志框架之间的差异。例如应用中使用如Log4j + SLF4J 这样的组合来进行日志输出；
  10. 禁止把SimpleDateFormat定义成static变量； => 线程不安全；
  11. 禁止开发人员使用isSuccess作为变量名； => Java Bean Specification规范，一般的属性用get/set, 布尔值属性用is/set, 因此is 会造成一些框架解析出错；
  12. 禁止开发人员修改serialVersionUID字段的值；=> 会造成反序列化出错；
  13. 建议开发者谨慎使用继承； => 扩展问题，接口优于继承；
  14. 禁止使用count(列名) 或count(常量) 来替代count(*)；=> 性能问题，差距很大；
